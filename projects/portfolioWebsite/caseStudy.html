Case Study: Portfolio Website Project

Project Overview

This project involved creating a portfolio website to showcase professional skills, projects, and articles. The website was developed with a focus on core functionality, essential content, and establishing design patterns, prioritizing simplicity and function over form. The goal was to create a solid foundation for future development.

Project Brief

Project Brief

Development Process

Planning

Total planning time: 1 hour

  • Defined achievable core requirements that will serve as the foundation for future development iterations of the website. These sections provide a comprehensive overview of professional identity and work.
  • Decided on a minimalist approach using only HTML for the initial version. This was done to keep focus on content structure and rapid deployment.
  • Defined a formal project creation process. By creating a project brief and case study, the development process and design choices of each project are documented for future analysis.

Development

Total development time: 2.5 hours

Key Findings

  1. Content Rendering
    Challenge: How to display Markdown content on the website
    Process: Researched methods for rendering Markdown content

    • Considered client-side rendering with JavaScript libraries

    • Opted for pre-converting Markdown to HTML

    Outcome: Better SEO and simpler initial implementation, aligning with the goal of a solid foundation

  2. Image Generation for Articles
    Challenge: Finding appropriate images for articles
    Process: Used AI (Gemini) to generate custom images. Images aren't perfect but that adds to the appeal
    Outcome: Avoided copyright concerns and created unique, relevant visuals efficiently

  3. Using AI for Development
    Challenge: Extracting development value by prompting an LLM
    Process: Wrote HTML and content using Claude.ai due to familiarity
    Outcome: Leveraged AI to draft content quickly, allowing focus on overall structure and implementation

  4. Website Hosting
    Challenge: How to host the website on the Internet
    Process: Elected to host on Github Pages due to it being free and existing familiarity
    Outcome: Website became publically available

Technical Details

  • Languages Used: HTML
  • Content Format: Markdown
  • Hosting: GitHub Pages

Results

  • Successfully created a basic portfolio website meeting all initial requirements
  • Established a foundation for future enhancements and additions
  • Gained experience in using AI tools for content generation and problem-solving
  • Recognized the value of starting with a simple, achievable scope

Future Enhancements

  • Use a front-end framework to enhance styling and layout
  • Add TypeScript for interactive elements
  • Expand the Articles and Projects sections with more content
  • Implement a more sophisticated method for managing and displaying Markdown content

Reflections

  • Starting simple allows for faster initial deployment
  • Even simple projects consume more time than anticipated
  • Planning and content creation can be more challenging than coding
  • AI can be a valuable tool in the development process

Conclusion

This project served as an excellent starting point for creating an online presence. By focusing on essential content and a solid HTML structure, it laid the groundwork for future improvements and expansions. The process provided valuable insights into efficient web development practices and the effective use of AI tools in the development workflow.
